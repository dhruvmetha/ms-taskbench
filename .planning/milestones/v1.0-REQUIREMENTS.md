# Requirements: Program Synthesis Bed

**Defined:** 2026-02-13
**Core Value:** Reliably execute multi-step pick-place sequences on configurable N-cube environments and produce video demos

## v1 Requirements (All Shipped)

### Environment

- [x] **ENV-01**: User can specify number of cubes N via `gym.make("StackNCube-v1", num_cubes=N)` — Validated: Phase 1
- [x] **ENV-02**: N cubes spawn on table with collision-free random placement each episode — Validated: Phase 1
- [x] **ENV-03**: Success evaluation checks all N-1 adjacent pairs are stacked, static, and released — Validated: Phase 1
- [x] **ENV-04**: Episode step budget scales with N to prevent premature truncation — Validated: Phase 1 (250 steps default)

### Skill

- [x] **SKIL-01**: Skill chains N-1 pick-place operations sequentially in a single episode — Validated: Phase 2
- [x] **SKIL-02**: Each pick-place targets the current stack-top position (dynamic height computation) — Validated: Phase 2
- [x] **SKIL-03**: Episode aborts immediately if any grasp or place step fails — Validated: Phase 2

### Integration

- [x] **INTG-01**: `num_cubes` configurable via Hydra (`env.num_cubes=5`) — Validated: Phase 3
- [x] **INTG-02**: `run.py` dispatches to new stacking skill via `run.policy=stack_n` — Validated: Phase 3
- [x] **INTG-03**: Video recording captures full multi-cube stacking sequence — Validated: Phase 3

## v2 Requirements (Deferred)

### Robustness

- **ROB-01**: plan_pose (RRT) fallback when plan_screw fails near obstacles
- **ROB-02**: Planner scene awareness via update_point_cloud for placed cubes
- **ROB-03**: Partial success metrics (cubes_stacked count in evaluate())
- **ROB-04**: Failure recovery/retry per cube before aborting
- **ROB-05**: Stacking order specification (user-defined sequence)

## Out of Scope

| Feature | Reason |
|---------|--------|
| RL policy for N-cube stacking | Research problem, not demo generation scope |
| GPU-parallel motion planning | mplib requires num_envs=1, CPU single-env |
| Dynamic N mid-episode | ManiSkill3 actors loaded once at construction via _load_scene |
| Heterogeneous object shapes | Scope creep — cubes only for this milestone |
| Configurable cube sizes | Separate concern, defer to later milestone |
| Dense reward shaping for N cubes | Only needed for RL, not motion-planned demos |
